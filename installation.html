Installer Debian / Ubuntu Server
Debian/Ubuntu — 6 étapes
▼

La base la plus courante en homelab et en prod : installeur graphique simple, dépôts stables, énorme documentation en cas de blocage.

⌘ commandes Debian/Ubuntu

📥 Télécharger l'ISO officielle

  1. Télécharger l'image et vérifier son empreinte
    sha256sum debian-12.6.0-amd64-netinst.iso

    Compare le résultat avec le fichier SHA256SUMS officiel avant de graver quoi que ce soit — une ISO corrompue plante souvent au pire moment de l'install.

  2. Créer la clé USB bootable
    sudo dd if=debian-12.6.0-amd64-netinst.iso of=/dev/sdX bs=4M status=progress

    Sous Windows : Rufus. Vérifie bien /dev/sdX avec lsblk avant — une erreur ici écrase le mauvais disque.

  3. Démarrer et choisir un schéma de partitionnement

    Pour un serveur : LVM (Guided - use entire disk and set up LVM) plutôt que partitionnement manuel, ça facilite l'extension future avec lvextend.

  4. Sélectionner uniquement les rôles nécessaires

    Décoche "environnement de bureau" sur un serveur — ne garde que "SSH server" et "standard system utilities" pour rester léger.

  5. Premier démarrage : mise à jour immédiate du système
    sudo apt update && sudo apt full-upgrade -y
  6. Vérifications de base post-install
    ip a systemctl status ssh

    Confirme l'IP obtenue et que le service SSH tourne avant de débrancher l'écran/clavier.

🩹 Problèmes fréquents
La clé USB n'apparaît pas dans le menu de boot
Vérifie le mode Legacy/UEFI dans le BIOS — une clé créée en mode UEFI ne boot pas toujours en Legacy, et inversement.
Écran noir juste après le démarrage du live-USB (souvent avec un GPU Nvidia)
Au menu de boot, appuie sur e (ou Tab) et ajoute nomodeset à la ligne de démarrage avant de continuer.
Aucune carte réseau détectée pendant l'installation
Le firmware du contrôleur réseau manque souvent sur l'image "netinst" standard — retélécharge l'image incluant les firmware non-free.

Installer Alpine Linux
Alpine Linux — 5 étapes
▼

Le mode diskless (tout en RAM, rien sur disque tant qu'on ne commit pas) déroute au début — c'est un choix volontaire, pas un bug.

⌘ commandes Alpine

📥 Télécharger l'ISO officielle

  1. Télécharger l'image "standard" (pas "virt" sauf usage cloud/conteneur pur)
    sha256sum alpine-standard-3.20.0-x86_64.iso
  2. Créer la clé USB bootable
    sudo dd if=alpine-standard-3.20.0-x86_64.iso of=/dev/sdX bs=4M status=progress
  3. Lancer l'assistant d'installation une fois démarré
    setup-alpine

    Répond aux questions clavier/réseau/disque, puis choisis "sys" pour une installation classique sur disque (pas "disk" en mode diskless).

  4. Premier démarrage : mise à jour des paquets
    apk update && apk upgrade
  5. Vérifier que la configuration a bien été écrite sur le disque
    rc-status

    En mode diskless, n'oublie jamais lbu commit après un changement, sinon tout repart de zéro au redémarrage.

🩹 Problèmes fréquents
setup-alpine échoue au moment de choisir le disque
Vérifie d'abord que le disque cible existe bien avec fdisk -l ou lsblk avant de relancer l'assistant.
Toute la configuration disparaît après un redémarrage
C'est le mode diskless normal (tout vit en RAM) — il faut lancer lbu commit après chaque changement pour que ce soit sauvegardé sur le disque.
Pas d'environnement graphique après l'installation
Normal : l'image "standard" n'installe volontairement aucune interface graphique — il faut l'ajouter soi-même via apk add xorg-server ... si besoin.

Installer Arch Linux (manuel)
Arch Linux — 7 étapes
▼

Pas d'installeur graphique par défaut : chaque étape est une commande volontaire. Plus long, mais on comprend exactement ce qui a été fait — l'esprit même d'Arch.

⌘ commandes Arch Linux

📥 Télécharger l'ISO officielle

  1. Vérifier la connectivité réseau depuis le live-ISO
    ping -c 3 archlinux.org
  2. Partitionner le disque
    cfdisk /dev/sda

    Au minimum : une partition EFI (512 Mo, type EFI System) et une partition racine.

  3. Formater et monter les partitions
    mkfs.ext4 /dev/sda2 mount /dev/sda2 /mnt
  4. Installer les paquets de base
    pacstrap /mnt base linux linux-firmware vim
  5. Générer la table des systèmes de fichiers
    genfstab -U /mnt >> /mnt/etc/fstab
  6. Entrer dans le nouveau système et finaliser (bootloader, mot de passe root, hostname)
    arch-chroot /mnt

    C'est dans ce chroot qu'on installe GRUB, qu'on définit passwd et hostnamectl avant de redémarrer.

  7. Premier démarrage : mise à jour du système
    pacman -Syu
🩹 Problèmes fréquents
pacstrap échoue avec une erreur de clé de signature (archlinux-keyring)
Rafraîchis les clés avant de continuer : pacman -Sy archlinux-keyring puis relance pacstrap.
Le système ne démarre plus après le premier redémarrage
Vérifie que /boot/efi était bien monté avant grub-install, et que grub-mkconfig -o /boot/grub/grub.cfg a bien été exécuté dans le chroot.
Pas de réseau une fois dans le système fraîchement installé
Les services réseau ne sont pas activés par défaut après un pacstrap minimal — active systemctl enable --now NetworkManager (ou systemd-networkd).

Installer RHEL / Rocky / AlmaLinux
RHEL — 6 étapes
▼

Anaconda (l'installeur graphique) est bien plus complet que celui de Debian — SELinux et le partitionnement avancé se configurent dès l'install, autant en profiter.

⌘ commandes RHEL

📥 Télécharger l'ISO officielle

  1. Vérifier l'empreinte de l'image téléchargée
    sha256sum -c CHECKSUM
  2. Créer le support d'installation
    sudo dd if=Rocky-9-x86_64-minimal.iso of=/dev/sdX bs=4M status=progress
  3. Choisir le schéma de partitionnement dans Anaconda

    "Automatic" avec LVM convient à la plupart des cas ; en prod, sépare souvent /var et /home sur des volumes logiques distincts.

  4. Vérifier que SELinux est bien actif après l'installation
    sestatus

    SELinux "Enforcing" par défaut sur RHEL — ne le désactive pas au moindre blocage, résous plutôt le contexte avec restorecon/semanage.

  5. Premier démarrage : mise à jour et enregistrement de l'abonnement (RHEL uniquement)
    sudo dnf update -y subscription-manager register
  6. Ouvrir les ports nécessaires dans le pare-feu
    firewall-cmd --permanent --add-service=ssh && firewall-cmd --reload
🩹 Problèmes fréquents
Un service refuse de démarrer sans message d'erreur clair
Le coupable est souvent SELinux plutôt qu'une vraie erreur de config — vérifie avec ausearch -m avc -ts recent avant de désactiver quoi que ce soit.
subscription-manager register échoue
Vérifie la connectivité vers le Customer Portal Red Hat et l'exactitude des identifiants — un proxy d'entreprise bloque parfois cette connexion.
Plus assez d'espace sur / après l'installation
Étends le volume logique à chaud : lvextend -L +10G /dev/mapper/rhel-root puis xfs_growfs /.

Installer FreeBSD
FreeBSD — 5 étapes
▼

bsdinstall est un installeur en mode texte, simple une fois qu'on a compris que ZFS est presque toujours le bon choix de système de fichiers par défaut.

⌘ commandes FreeBSD

📥 Télécharger l'ISO officielle

  1. Créer le support d'installation
    sudo dd if=FreeBSD-14.0-RELEASE-amd64-dvd1.iso of=/dev/sdX bs=4M status=progress
  2. Lancer l'installeur et choisir le partitionnement
    bsdinstall

    "Auto (ZFS)" est recommandé par défaut : snapshots et rollback facilités dès le départ.

  3. Configurer le réseau pendant l'installation

    DHCP suffit pour un premier essai ; une IP fixe se configure ensuite via /etc/rc.conf.

  4. Premier démarrage : mise à jour du système de base
    freebsd-update fetch install
  5. Installer le gestionnaire de paquets et un premier outil
    pkg install vim
🩹 Problèmes fréquents
pkg install échoue avec "unable to update repository"
Vérifie /etc/pkg/FreeBSD.conf et la connectivité réseau ; un pkg update -f force la resynchronisation du catalogue.
Pas de réseau après l'installation
Vérifie le nom exact de l'interface avec ifconfig (souvent em0 ou re0, pas eth0) et corrige /etc/rc.conf en conséquence.
Le système ne démarre pas après une install ZFS
Vérifie que le bootloader ZFS a bien été installé sur le bon disque pendant bsdinstall — un oubli fréquent en partitionnement manuel.

Installer Windows 11
Windows — 5 étapes
▼

Le Media Creation Tool officiel évite les images modifiées qui traînent ailleurs — surtout pour une VM de test ou un lab qu'on veut fiable.

⌘ commandes Windows

📥 Télécharger l'ISO officielle

  1. Créer le support avec l'outil officiel Microsoft

    Media Creation Tool → "Créer un support d'installation" → clé USB. Fonctionne aussi pour générer directement un fichier ISO.

  2. Démarrer et choisir "Installation personnalisée"

    Pas "Mise à niveau" — repartir d'un disque vide évite d'hériter de résidus d'une install précédente.

  3. Contourner la vérification de compte Microsoft si besoin (compte local)
    OOBE\BYPASSNRO

    À taper dans l'invite de commande (Shift+F10) pendant l'installation pour forcer un compte local plutôt qu'un compte en ligne.

  4. Premier démarrage : vérifier les mises à jour Windows

    Paramètres → Windows Update → Rechercher des mises à jour, avant d'installer quoi que ce soit d'autre.

  5. Ouvrir un terminal PowerShell en administrateur pour la suite
    Get-ComputerInfo | Select WindowsProductName, OsVersion
🩹 Problèmes fréquents
"Cette clé USB ne peut pas être utilisée pour démarrer sur cet ordinateur"
Le mode de partition (GPT/UEFI vs MBR/Legacy) de la clé ne correspond pas au mode configuré dans le BIOS — bascule l'un des deux pour qu'ils correspondent.
Écran bleu INACCESSIBLE_BOOT_DEVICE pendant l'installation
Active le mode AHCI pour le contrôleur de stockage dans le BIOS avant de relancer l'installation (RAID/IDE posent souvent ce problème).
Pas de Wi-Fi disponible pendant l'installation pour se connecter à un compte Microsoft
Utilise un câble Ethernet, ou termine l'installation en compte local via OOBE\BYPASSNRO et configure le Wi-Fi après coup.

Installer Windows Server (Core ou Desktop Experience)
Windows Server — 6 étapes
▼

Le choix "Core" (sans interface graphique) économise des ressources et réduit la surface d'attaque — tout se pilote alors en PowerShell ou à distance.

⌘ commandes Windows Server

📥 Télécharger l'ISO officielle

  1. Démarrer l'installeur et choisir l'édition

    "Server Core" pour un rôle serveur classique (AD, fichiers, DHCP) ; "Desktop Experience" seulement si une interface graphique est vraiment nécessaire.

  2. Définir le mot de passe administrateur au premier démarrage
  3. Configurer le réseau en IP fixe (recommandé pour un serveur)
    New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.1.10 -PrefixLength 24 -DefaultGateway 192.168.1.1
  4. Renommer la machine et redémarrer
    Rename-Computer -NewName "SRV-01" -Restart
  5. Installer les mises à jour
    Install-WindowsUpdate
  6. Ajouter le premier rôle serveur (exemple : Active Directory)
    Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools
🩹 Problèmes fréquents
L'installation du rôle AD DS échoue
Vérifie qu'un serveur DNS fonctionnel est déjà configuré (souvent lui-même) avant de promouvoir le serveur en contrôleur de domaine.
Impossible de rejoindre le domaine depuis une autre machine
Vérifie d'abord l'heure système des deux machines — Kerberos refuse l'authentification au-delà de quelques minutes de décalage.
Écran noir prolongé après le premier redémarrage post-installation
C'est normal pendant la phase de finalisation ("Getting devices ready") — patiente plusieurs minutes avant de forcer un redémarrage.

Réinstaller macOS via le mode de récupération
macOS — 5 étapes
▼

Pas d'ISO classique sur Mac : tout passe par le mode de récupération intégré, qui télécharge le système directement depuis les serveurs Apple.

⌘ commandes macOS

📥 Télécharger l'ISO officielle

  1. Redémarrer en mode de récupération

    Mac Apple Silicon : maintenir le bouton d'alimentation au démarrage. Mac Intel : Cmd+R au démarrage.

  2. Effacer le disque si c'est une réinstallation complète

    Utilitaire de disque → sélectionner le disque interne → Effacer (format APFS).

  3. Réinstaller macOS depuis le menu de récupération

    Le système se télécharge directement depuis Apple — compter 30 à 60 minutes selon la connexion.

  4. Premier démarrage : installer les outils en ligne de commande
    xcode-select --install
  5. Installer Homebrew pour la gestion de paquets
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
🩹 Problèmes fréquents
Erreur réseau pendant le téléchargement en mode récupération
Réessaie sur un autre réseau — certains pare-feux/DNS d'entreprise bloquent les serveurs de récupération Apple.
Le Mac ne démarre pas en mode récupération
La combinaison diffère selon le modèle : bouton d'alimentation maintenu sur Apple Silicon, Cmd+R au démarrage sur Intel.
"brew: command not found" après l'installation de Homebrew
Sur Apple Silicon, Homebrew s'installe dans /opt/homebrew et pas /usr/local — ajoute-le au PATH avec la commande affichée en fin d'installation.

Installer Proxmox VE
Proxmox VE — 6 étapes
▼

Contrairement à Docker ou Kubernetes qui s'installent sur un OS existant, Proxmox VE est lui-même l'OS : un hyperviseur Debian qui prend toute la machine dès l'installation.

⌘ commandes Proxmox VE

📥 Télécharger l'ISO officielle

  1. Télécharger l'ISO et vérifier son empreinte
    sha256sum proxmox-ve_9.2-1.iso
  2. Créer la clé USB bootable
    sudo dd if=proxmox-ve_9.2-1.iso of=/dev/sdX bs=4M status=progress
  3. Choisir le système de fichiers pendant l'installation

    ZFS (RAID1/RAIDZ selon le nombre de disques) si tu as au moins 2 disques et de la RAM à revendre ; ext4 sur LVM-thin sinon, plus simple sur un seul disque.

  4. Configurer le réseau du nœud pendant l'installation

    Une IP fixe est fortement recommandée — l'interface web et la gestion du cluster en dépendent directement.

  5. Se connecter à l'interface web une fois redémarré
    https://IP-DU-SERVEUR:8006
  6. Basculer sur le dépôt gratuit "no-subscription"
    echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" > /etc/apt/sources.list.d/pve-no-subscription.list apt update && apt full-upgrade -y

    Sans ça, apt update échoue en cherchant le dépôt "enterprise" qui nécessite un abonnement payant.

🩹 Problèmes fréquents
apt update échoue en cherchant le dépôt "enterprise"
Normal sans abonnement payant — bascule sur le dépôt pve-no-subscription comme indiqué à l'étape 6.
L'interface web sur le port 8006 ne répond pas
Vérifie l'IP configurée pendant l'installation et qu'aucun pare-feu intermédiaire ne bloque le port 8006.
Avertissement ZFS sur l'absence de RAM ECC
C'est un avertissement, pas un blocage — ZFS fonctionne sans ECC, avec juste un risque (minime) de corruption silencieuse en cas d'erreur mémoire non détectée.
